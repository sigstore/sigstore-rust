# Test fixtures

These TUF metadata fixtures are vendored from the
[`tough`](https://github.com/awslabs/tough) project's test data
(`tough/tests/data/`), which is licensed `MIT OR Apache-2.0` —
Copyright Amazon.com, Inc. or its affiliates.

- `tuf-reference-impl/` — a complete repository **generated by `python-tuf`**
  (the TUF reference implementation), with two levels of targets delegation
  (`role1` → `role2`) and targets carrying `custom` metadata. We use it to prove
  that `sigstore-tuf` interoperates byte-for-byte with metadata produced by
  `python-tuf`: full refresh, target download, and delegation resolution.
- `negative/*.root.json` — individual malformed `root.json` files used to check
  that bootstrap fails closed:
  - `no-root-json-signatures` — zero signatures (threshold not met),
  - `invalid-root-json-signature` — a corrupted signature,
  - `duplicate-sigs` — two signatures from the *same* key against a threshold of
    two (must not satisfy the threshold).

They are unmodified copies; see `tough`'s repository for how they were generated.
